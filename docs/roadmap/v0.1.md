## 3. Release stages

按 [PRD § 13](../bmad/02-planning/PRD.md) 4 stages：

### 3.1 Dogfood（仅作者）

**入门条件**：13 story 全部 done + 实测全过

**标准**：
- 作者自己 **14 天连续使用不删**
- 每天打开 popup ≥ 5 次
- 至少 3 次"靠它发现某个 session 等我已超过 5 分钟"实际事件
- 完成 retrospective（用 [dogfood-retrospective-template.md](../guides/dogfood-retrospective-template.md)）

**SLA**：
- 14 天内 0 次 crash
- 24h RSS 增长 < 50MB
- 空闲 CPU < 0.5% (M1)

**退出 = 进入 closed alpha**

### 3.2 Closed alpha（5-10 朋友）

**入门条件**：作者 dogfood retrospective 写完 + P0 bug 修完

**标准**：
- 5-10 个 Heavy CC user 朋友收到 DMG + install.md
- 跟每人 1 对 1 聊一次（30 min）收集反馈
- 至少 4/5 用户能完成 [scenarios.md § S4 onboarding](../product/scenarios.md) 30 min 内 "啊有用" 瞬间
- 至少 4/5 用户 14 天后仍在用（D14 retention）

**退出 = 进入 public beta**

### 3.3 Public beta

**入门条件**：alpha 反馈处理完 + install.md 跨 macOS 版本实测

**渠道**（详 [launch-plan.md](launch-plan.md)）：
- GitHub Release v0.1.0 (DMG + checksums)
- 推特发布
- Anthropic Discord 社区

**标准**：
- 30 天内 GitHub stars ≥ 10
- 30 天内收到 ≥ 3 个 issue from 真实用户（非作者）

**退出 = 进入 v1.0**

### 3.4 v1.0

**入门条件**：beta 期 30 天无 P0/P1 issue

**渠道**：
- Show HN
- Homebrew cask 提交
- 中文技术媒体推广

**标准**：
- 90 天 GitHub stars ≥ 100
- 至少 1 个外部 contributor PR merge
- Anthropic 官方提及（aspirational）
